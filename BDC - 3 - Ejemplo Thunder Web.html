Análisis de Situación

  • Fortaleza Cultural: Amplio uso de datos en Finanzas, Ventas y Logística, lo que facilita la transición.
  • Infraestructura: SAP BW 7.5 on HANA es robusto pero tradicional. La arquitectura actual es monolítica y requiere evolución hacia la federación.
  • Gobierno del Dato: Aunque hay seguridad básica (roles), falta trazabilidad "End-to-End" y un catálogo de datos unificado.
  • Fragmentación: Uso de múltiples herramientas de consumo (Analyzer, Board, Excel manual) que generan silos de información.
  • Oportunidad de Costes: Elevado coste de mantenimiento de sistemas legacy y licencias dispersas.

Arquitectura TO-BE Detallada

1. Capa de Consumo Inteligente

Punto único de acceso mediante SAP Analytics Cloud, Apps Inteligentes y el Add-In para Office. Visualización unificada sin silos.

2. Capa Semántica (SAP Datasphere)

El corazón del sistema. Gestiona la federación, el modelado de negocio, el Catálogo de Datos y el Linaje. Aquí reside la "verdad única".

3. Capa de IA & Ciencia de Datos

Integración con Databricks para modelos complejos de Machine Learning, predicción de demanda y algoritmos avanzados.

4. Fuentes de Datos (Tierras)

Conectores nativos para SAP S/4HANA, SuccessFactors, Ariba y fuentes externas (Azure, AWS, Archivos Planos).

Modelo de Gobierno y Calidad

Basado en las mejores prácticas DMBOK (Data Management Body of Knowledge)

Catálogo y Glosario

Definición de términos de negocio y diccionario de datos compartido para evitar ambigüedades.

Linaje del Dato

Trazabilidad completa (End-to-End) para saber de dónde viene y quién usa cada dato.

Calidad (Data Quality)

Reglas automáticas para detectar duplicados, nulos e inconsistencias antes de que lleguen al reporte.

Seguridad

Roles RBAC/ABAC centralizados. Cumplimiento normativo (GDPR) y auditoría de accesos.

Organización

Definición de Data Stewards y Comités de Gobierno para la gestión del cambio.

Arquitectura

Estándares de integración y modelos de datos canónicos.
